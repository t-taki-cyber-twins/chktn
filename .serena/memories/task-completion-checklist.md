# タスク完了時のチェックリスト

## コード品質チェック

### ビルド確認
- [ ] `./gradlew build` が成功する
- [ ] コンパイルエラーがない
- [ ] 警告がない（または許容範囲内）

### テスト
- [ ] `./gradlew test` が成功する
- [ ] 新しく追加した機能のテストが含まれている
- [ ] 既存のテストが壊れていない

## Docker Compose動作確認

### サービス起動確認
- [ ] `docker compose up -d` でサービスが起動する
- [ ] すべてのサービスが正常に起動している（`docker compose ps`）
- [ ] ログにエラーが出ていない（`docker compose logs`）

### API動作確認
- [ ] API Gateway（8080）にアクセスできる
- [ ] Core Service（8081）にアクセスできる
- [ ] Swagger UIが表示される（http://localhost:8081/swagger-ui.html）
- [ ] 基本的なAPIリクエストが成功する

## データベース確認

### マイグレーション
- [ ] Flywayマイグレーションが正常に実行されている
- [ ] 新しいマイグレーションファイルがあれば適用されている
- [ ] マイグレーション履歴が正しい（`SELECT * FROM flyway_schema_history;`）

### データ整合性
- [ ] テーブル構造が正しい
- [ ] 必須カラム（`tenant_id`等）がすべてのテーブルに存在する

## コード規約確認

### 命名規則
- [ ] クラス名が命名規則に従っている
- [ ] パッケージ構造が正しい
- [ ] データベーステーブル・カラム名がスネークケース

### アーキテクチャ
- [ ] レイヤー構成が正しい（Controller → Service → Repository）
- [ ] DTOとエンティティが適切に分離されている
- [ ] マルチテナント対応が実装されている（該当する場合）

### OpenAPI仕様
- [ ] `@Tag`、`@Operation`、`@ApiResponse`が適切に設定されている
- [ ] Swagger UIでAPI仕様が正しく表示される

## ドキュメント

### コードドキュメント
- [ ] JavaDocコメントが適切に記載されている
- [ ] 複雑なロジックにコメントがある

### プロジェクトドキュメント
- [ ] 重要な変更があれば`docs/`フォルダのドキュメントを更新した
- [ ] API仕様の変更があれば`apidog-config.yaml`を更新した（該当する場合）

## 最終確認

### 動作確認
- [ ] 実装した機能が期待通りに動作する
- [ ] エラーハンドリングが適切に実装されている
- [ ] パフォーマンスに問題がない（簡単な確認）

### コミット前
- [ ] 不要なファイル（ログ、一時ファイル等）が含まれていない
- [ ] `.env`ファイルがコミットされていない（Git除外されている）
- [ ] コミットメッセージが適切に記載されている
