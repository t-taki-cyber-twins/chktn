# アプリケーションヘッダー (app-header.js)

## 概要
アプリケーション全体の共通ヘッダーコンポーネントです。各機能へのナビゲーション、ユーザー情報の表示、ログアウト機能などを提供します。

## 機能の目的
- **ナビゲーション**: ユーザーが目的の機能（メッセージ、エンジニア管理、案件管理など）へ素早くアクセスできるようにする。
- **コンテキスト表示**: 現在ログインしているユーザー名、テナント名、会社名を表示し、ログイン状態を明確にする。
- **システム管理へのアクセス**: 権限に応じて、テナント管理やアカウント管理などの管理機能へのアクセスポイントを提供する。

## ユースケース
- **機能間の移動**: ユーザーは、ヘッダーのメニューから「メッセージ検索」や「案件検索」などの各画面へ移動する。
- **管理画面へのアクセス**: 管理者は、ヘッダーのアイコンから「アカウント管理」や「テナントマスタ管理」メニューを開き、設定画面へ移動する。
- **ユーザー情報の確認**: ユーザーは、右上のユーザー名をクリックして、自身の所属テナントや会社名を確認する。
- **ログアウト**: ユーザーは、ユーザーメニューからログアウトを行う。

## EARS 記法による要件定義

### 表示・ナビゲーション
- **WHEN** 画面が表示された際
  - **THEN** システムはヘッダーを画面上部に固定表示する
  - **AND** ロゴ、各機能へのナビゲーションメニュー、管理者用アイコン、ユーザー名を表示する

- **WHEN** ユーザーがロゴをクリックした際
  - **THEN** システムはトップ画面（`top.html`）へ遷移する

### ドロップダウンメニュー
- **WHEN** ユーザーがナビゲーションメニューまたはアイコンをクリックした際
  - **THEN** システムは対応するドロップダウンメニューを表示する
  - **AND** 他の開いているドロップダウンメニューを閉じる

- **WHEN** ユーザーがドロップダウンメニュー内の項目をクリックした際
  - **THEN** システムは選択された機能の画面へ遷移する

### ユーザー情報・ログアウト
- **WHEN** ユーザーがユーザー名をクリックした際
  - **THEN** システムはユーザー情報（テナント名、会社名）とログアウトボタンを含むメニューを表示する

- **WHEN** ユーザーが「ログアウト」をクリックした際
  - **THEN** システムはログアウト処理を行い、ログイン画面（`index.html`）へ遷移する

## ユーザ確認事項
- **権限による表示制御**: 現在の実装ではすべてのメニューが表示されていますが、ユーザーの権限（ロール）に応じて表示するメニューを制御する必要があるか確認が必要です。
- **未実装リンク**: 「エンジニア一括登録」「案件一括登録」「お知らせ管理」「リファラー管理」などのリンクが `#` となっています。これらの実装予定や遷移先の確認が必要です。
