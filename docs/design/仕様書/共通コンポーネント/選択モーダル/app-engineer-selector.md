# エンジニア選択モーダル (app-engineer-selector.js)

## 概要
システム内のエンジニアを検索・選択するための共通モーダルコンポーネントです。
案件へのエンジニア割り当てや、エンジニア情報の参照時に使用されます。
マッチングスコアの表示にも対応しており、案件に最適なエンジニアを選定する場面で活用されます。

## 機能の目的
- ユーザーが条件に合致するエンジニアを効率的に検索・選択できるようにする。
- スキルやマッチングスコア（案件との適合度）を可視化し、適切な人材選定を支援する。

## ユースケース
- **公開案件へのエンジニア提案**: 公開案件詳細画面にて、その案件に提案する自社エンジニアを選択する。
- **エンジニア情報の参照**: 特定のエンジニアを選択して詳細情報を確認する（将来的な拡張）。

## EARS 記法による要件定義

### モーダル表示・初期化
- **WHEN** 呼び出し元画面からエンジニア選択モーダルが開かれた際
  - **THEN** システムはエンジニア一覧を表示する
  - **IF** URLパラメータ等で初期選択エンジニアが指定されている場合
    - **THEN** システムは該当するエンジニアを選択状態（ハイライト）で表示する

### 検索・フィルタリング
- **WHEN** ユーザーが検索条件（エンジニア名、スキル）を入力して「検索」ボタンをクリックした際
  - **THEN** システムは条件に一致するエンジニアのみをフィルタリングして表示する

- **WHEN** ユーザーが「リセット」ボタンをクリックした際
  - **THEN** システムは検索条件をクリアし、全エンジニアを表示する
  - **AND** 選択中のエンジニアがあれば選択を解除する

### 選択・確定
- **WHEN** ユーザーがエンジニアリストの項目をクリックした際
  - **THEN** システムはクリックされたエンジニアを選択状態（ハイライト）にする
  - **AND** 「選択」ボタンを有効化（活性化）する
  - **AND** 他に選択されていたエンジニアがあれば選択を解除する（単一選択）

- **WHEN** ユーザーが「選択」ボタンをクリックした際
  - **THEN** システムは選択されたエンジニア情報を呼び出し元に返し、モーダルを閉じる

- **WHEN** ユーザーが「キャンセル」または「×」ボタンをクリックした際
  - **THEN** システムは変更を破棄してモーダルを閉じる

## 機能一覧

| コンポーネント | 機能名 | 概要 | 優先度 | アクター | 備考 |
|---|---|---|---|---|---|
| エンジニア選択モーダル | エンジニア一覧表示 | エンジニアをリスト表示する | 必須 | システム | 名前、スキルタグ、マッチングスコア（ある場合）を表示 |
| エンジニア選択モーダル | エンジニア検索 | エンジニア名（部分一致）、保有スキルで検索する | 必須 | ユーザー | スキルは複数選択可能 |
| エンジニア選択モーダル | 検索リセット | 検索条件と選択状態を初期化する | 必須 | ユーザー | |
| エンジニア選択モーダル | 単一選択 | リストからエンジニアを1つ選択する | 必須 | ユーザー | 行クリックで選択 |
| エンジニア選択モーダル | 選択確定 | 選択したエンジニア情報を確定し、呼び出し元へ渡す | 必須 | ユーザー | |
| エンジニア選択モーダル | マッチングスコア表示 | 案件との適合度（スコア）を表示する | 必須 | システム | スコアに応じて色分け表示（高・中・低） |

## ユーザ確認事項
- **スキル検索のロジック**: 複数のスキルを選択した場合の検索ロジック（AND条件かOR条件か）について確認が必要です。現状は簡易実装となっています。
- **ページネーション**: エンジニア数が増加した場合の対応（ページネーションまたは無限スクロール）について確認が必要です。
- **表示項目**: 一覧に表示する項目（現在は名前、スキル、スコア）で十分か、他に必要な情報（稼働状況、単価など）がないか確認が必要です。
