# 公開エンジニア詳細 (public-engineer-detail.html)

## 概要
他社が公開しているエンジニアの詳細情報を閲覧し、スカウト（面談申込）や問い合わせを行うための画面です。
エンジニアのスキルや経歴を詳細に確認した上で、自社の案件にアサインするためのアクションを起こすことができます。

## 機能の目的
- エンジニアの詳細情報を提示し、自社案件への適合性を判断する材料を提供する。
- スカウトや問い合わせといった、エンジニア調達のための具体的なアクションを可能にする。

## ユースケース
- **エンジニア詳細の確認**: ユーザーは、エンジニアの自己紹介、スキル詳細、経歴、希望条件などを確認する。
- **スカウト（面談申込）**: ユーザーは、自社の案件を選択し、その案件に対してエンジニアをスカウト（面談申込）する。
- **問い合わせ**: ユーザーは、エンジニアに関する不明点などを問い合わせる。
- **メッセージ履歴の確認**: ユーザーは、このエンジニアに関する過去の問い合わせややり取りの履歴を確認する。

## EARS 記法による要件定義

### 詳細表示・アクション
- **WHEN** ユーザが公開エンジニア詳細画面を開いた際
  - **THEN** システムはエンジニアの詳細情報（基本情報、スキル、経歴など）を表示する
  - **AND** 過去の問い合わせスレッド一覧を表示する
  - **AND** 自社との関連案件（面談中・稼働中）があれば一覧表示する

- **WHEN** ユーザがサイドバーの「案件を選択」ボタンをクリックした際
  - **THEN** システムは案件選択モーダル（`app-project-selector`）を表示する

- **WHEN** ユーザが案件を選択し、「面談申込」ボタンをクリックした際
  - **THEN** システムは確認ダイアログを表示する
  - **IF** ユーザが承認した場合
    - **THEN** システムは面談申込処理（スカウト）を実行し、完了メッセージを表示する

- **WHEN** ユーザが「問い合わせ」ボタンをクリックした際
  - **THEN** システムは問い合わせ入力フォーム（またはモーダル）を表示する（※現状はアラートのみ）

## ユーザ確認事項
- **スカウトフロー**: 現状は「面談申込」ボタンクリックで即座にスカウト送信（確認ダイアログ経由）となっていますが、メッセージを添えたり、面談候補日を提示したりする画面を挟む必要があるか確認が必要です。
- **問い合わせUI**: 「問い合わせ」ボタンクリック時の挙動が現状アラートのみです。専用の入力モーダルか、メッセージスレッドへの新規追加フローを明確にする必要があります。
- **関連案件の表示**: 「表示しているエンジニアのかつ自テナントの案件」を表示するセクションがありますが、これは既にコンタクトがある場合に有用です。新規のエンジニアの場合は表示されない前提でよいか確認が必要です。
