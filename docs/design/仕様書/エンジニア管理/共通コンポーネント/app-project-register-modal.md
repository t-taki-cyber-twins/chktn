# 案件登録モーダル (app-project-register-modal.js)

## 概要
新しい案件情報を登録するためのモーダルコンポーネントです。
エンジニア編集画面などから、関連する案件を即座に新規登録する場合などに使用されます。

## 機能の目的
- 画面遷移することなく、その場で新しい案件情報を登録できるようにする。
- 案件に必要な詳細情報（基本情報、関連企業、スキル、契約条件など）を網羅的に入力可能にする。

## ユースケース
- **案件の新規登録**: ユーザーは、エンジニアの経歴入力中や面談設定中に、システムに存在しない案件を新規登録する。

## EARS 記法による要件定義

### モーダル制御
- **WHEN** 親コンポーネントが `open()` メソッドを呼び出した際
  - **THEN** システムは案件登録モーダルを表示する

- **WHEN** ユーザが「キャンセル」または「×」ボタンをクリックした際
  - **THEN** システムはモーダルを閉じる

### 案件情報入力
- **WHEN** ユーザが案件情報を入力し「登録」ボタンをクリックした際
  - **THEN** システムは入力内容を検証する
  - **IF** 入力が有効な場合
    - **THEN** システムは案件情報を保存する（現状はモック）
    - **AND** モーダルを閉じる

## ユーザ確認事項
- **コンポーネントの再利用**: ブラックリスト選択やメーリングリスト選択などが、`app-blacklist-selector` などの既存コンポーネントを使用せず、独自に実装されているように見受けられます。共通コンポーネントへの置き換えを検討する必要があります。
- **登録後の挙動**: 登録完了後、呼び出し元の画面（エンジニア編集画面など）にどのようにデータを渡すか（コールバックやイベント発火など）の仕様が明確ではありません。
