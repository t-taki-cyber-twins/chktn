# おすすめエンジニア検索 (recommended-engineer-search.html)

## 概要
システムが自動的にマッチングした「公開エンジニア」と「自社案件」の組み合わせを検索・閲覧するための画面です。
マッチングスコアや理由を確認し、確度の高いエンジニアに対して効率的にアプローチすることができます。

## 機能の目的
- 膨大なエンジニア情報の中から、自社案件に適した人材を自動的に抽出し、採用・調達の機会損失を防ぐ。
- マッチングスコアや理由を提示することで、スカウトの判断を支援する。

## ユースケース
- **おすすめエンジニアの確認**: ユーザーは、システムが推奨するエンジニアと案件の組み合わせを確認する。
- **条件による絞り込み**: ユーザーは、エンジニア名、スキル、案件名などの条件で結果を絞り込む。
- **スカウト（面談申込）**: ユーザーは、マッチング結果から直接、または詳細画面を経由して面談を申し込む。

## EARS 記法による要件定義

### マッチング検索・一覧表示
- **WHEN** ユーザがおすすめエンジニア検索画面を開いた際
  - **THEN** システムは公開エンジニアと自社案件のマッチング結果をスコア順に表示する
  - **AND** 検索フォームを表示する

- **WHEN** ユーザが検索条件（エンジニア名、スキル、案件名など）を入力して「検索」ボタンをクリックした際
  - **THEN** システムは条件に一致するマッチング結果のみをフィルタリングして表示する

- **WHEN** ユーザが並び順（スコア順、エンジニア名順、稼働開始日順）を変更した際
  - **THEN** システムは指定された順序でリストを再描画する

### アクション実行
- **WHEN** ユーザが「面談申込」ボタンをクリックした際
  - **THEN** システムは公開エンジニア詳細画面へ遷移し、該当案件が選択された状態で面談申込フローを開始する（パラメータ渡し）

- **WHEN** ユーザが「詳細」ボタンをクリックした際
  - **THEN** システムは公開エンジニア詳細画面へ遷移する

## ユーザ確認事項
- **マッチングロジックの精度**: 現状はモック（ランダムスコア）ですが、実運用におけるマッチングアルゴリズム（スキル一致度、稼働時期、単価感など）の具体的な仕様策定が必要です。
- **表示情報の範囲**: エンジニア情報としてイニシャルやスキルを表示していますが、スカウト判断に十分な情報量か確認が必要です。
