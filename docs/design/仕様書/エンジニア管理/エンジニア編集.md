# エンジニア編集 (engineer-edit.html)

## 概要
既存のエンジニア情報を編集するための画面です。
基本情報やスキルの更新だけでなく、そのエンジニアに関連する案件（面談中・稼働中）の一覧表示や、面談情報の追加・編集もここから行うことができます。
処理のほとんどはエンジニア編集と共通で使用するapp-engineer-form.jsで行うことになる。

## 機能の目的
- 登録済みのエンジニア情報を最新の状態に更新する。
- エンジニアの稼働状況（案件アサイン状況）を把握し、面談調整などのアクションを行う。

## ユースケース
- **エンジニア情報の更新**: ユーザーは、エンジニアのスキルアップや稼働条件の変更に合わせて情報を更新する。
- **関連案件の確認**: ユーザーは、このエンジニアが現在面談中または稼働中の案件を確認する。
- **面談情報の管理**: ユーザーは、新たな案件への面談を追加したり、既存の面談情報を更新したりする。

## 機能一覧

| 画面 | 機能名 | 概要 | 優先度 | アクター | 備考 |
|---|---|---|---|---|---|
| エンジニア編集 | エンジニア情報ロード | 画面表示時に既存のエンジニア情報を取得し、フォームにセットする | 必須 | システム | |
| エンジニア編集 | エンジニア情報更新 | フォームの入力内容でエンジニア情報を更新する | 必須 | ユーザー | |
| エンジニア編集 | 関連案件一覧表示 | エンジニアに関連する案件（面談中、稼働中など）を一覧表示する | 必須 | システム | |
| エンジニア編集 | 面談追加 | 新たな面談情報を登録する | 必須 | ユーザー | `app-engineer-meeting-edit`を使用 |
| エンジニア編集 | 面談編集 | 既存の面談情報を編集する | 必須 | ユーザー | `app-engineer-meeting-edit`を使用 |
| エンジニア編集 | 案件編集遷移 | 関連案件一覧から案件編集画面へ遷移する | 必須 | ユーザー | 自社案件の場合 |

## ユーザ検討事項
- **関連案件の表示範囲**: 画面上部に表示するのは「表示している自テナントのエンジニアかつ面談中のものと稼働中の案件」です。過去の稼働履歴や、不採用になった案件は下の案件一覧で見れればOK。
