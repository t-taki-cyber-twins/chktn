# 取引先管理 (client-list.html)

## 概要
テナントの取引先会社を管理するための画面です。
取引先会社の追加・編集・削除を行い、案件やエンジニアの提供元・提供先として活用できるようにします。

## 機能の目的
- 取引先会社の情報を一元管理し、案件やエンジニアのマッチング時に正確な企業情報を活用する。
- 法人番号検索により、正確な企業情報を効率的に登録する。

## ユースケース
- **取引先の追加**: ユーザーは、新しい取引先会社を登録する（法人番号検索を利用可能）。
- **取引先の検索**: ユーザーは、会社名、法人番号、住所で取引先を検索する。
- **取引先情報の編集**: ユーザーは、既存の取引先会社の情報を更新する。
- **取引先の削除**: ユーザーは、不要になった取引先会社を削除する。

## EARS 記法による要件定義

### 取引先検索・一覧表示
- **WHEN** ユーザが取引先管理画面を開いた際
  - **THEN** システムは取引先会社の一覧を表示する
  - **AND** 検索フォームを表示する

- **WHEN** ユーザが検索条件（会社名、法人番号、住所）を入力して「検索」ボタンをクリックした際
  - **THEN** システムは条件に一致する取引先のみをフィルタリングして表示する

- **WHEN** ユーザが「リセット」ボタンをクリックした際
  - **THEN** システムは検索フォームの入力内容をクリアする
  - **AND** 全取引先の一覧を表示する

### 取引先管理
- **WHEN** ユーザが「＋ 取引先会社追加」ボタンをクリックした際
  - **THEN** システムは会社編集モーダル（`app-company-edit-modal`）を新規作成モードで表示する

- **WHEN** ユーザが「編集」ボタンをクリックした際
  - **THEN** システムは会社編集モーダルを編集モードで表示し、該当する取引先情報をロードする

- **WHEN** ユーザが「削除」ボタンをクリックした際
  - **THEN** システムは削除確認ダイアログを表示する
  - **IF** ユーザが承認した場合
    - **THEN** システムは取引先を削除し、一覧を更新する

## ユーザ確認事項
- **取引先と会社の違い**: 「tenant-company-list.html」では「会社」、こちらでは「取引先会社」となっていますが、内部的には同じ「会社」マスタとして管理されるのか、異なるエンティティとして管理されるのか確認が必要です。
- **法人番号検索の連携**: [app-official-company-search-modal](../共通コンポーネント/選択モーダル/app-official-company-search-modal.md)を利用した法人番号検索が想定されていますが、この検索結果を会社編集モーダルに自動入力する仕様の詳細確認が必要です。
