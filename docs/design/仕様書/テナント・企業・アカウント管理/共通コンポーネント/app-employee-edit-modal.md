# 社員情報編集モーダル (app-employee-edit-modal.js)

## 概要
社員情報の登録・編集を行うためのモーダルコンポーネントです。基本情報（氏名）と役割属性（エンジニア、案件担当、技術者管理）の設定を行い、詳細な社員情報は`app-employee-info-form`コンポーネントを使用して管理します。

## 機能の目的
- 社員の基本情報と役割（ロール）を一元管理する。
- 役割に応じた適切な権限や属性を付与する基盤とする。
- 既存の社員情報入力フォーム（`app-employee-info-form`）を再利用し、一貫性のある入力体験を提供する。

## ユースケース
- **新規社員の登録**: ユーザーは、新しく入社した社員の情報を登録し、適切な役割（案件担当など）を割り当てる。
- **社員情報の更新**: ユーザーは、社員の昇進や配置転換に伴い、役割属性を変更したり、所属部署情報を更新したりする。

## EARS 記法による要件定義

### モーダル表示
- **WHEN** 呼び出し元画面（社員一覧など）から「編集」または「新規登録」アクションが実行された際
  - **THEN** システムは社員情報編集モーダルを表示する
  - **IF** 既存データの編集の場合
    - **THEN** システムは現在の社員情報をフォームにプレ入力する
    - **AND** `app-employee-info-form` に詳細情報を設定する

### データ入力・保存
- **WHEN** ユーザーが役割属性（エンジニア、案件担当、技術者管理）のチェックボックスを変更した際
  - **THEN** システムは選択状態を保持する（UI上の即時変化のみ）

- **WHEN** ユーザーが「保存」ボタンをクリックした際
  - **THEN** システムは以下の情報を収集する
    - 基本情報（氏名）
    - 役割属性（エンジニア、案件担当、技術者管理）
    - 詳細情報（`app-employee-info-form`から取得）
  - **AND** 収集した情報を保存する
  - **AND** モーダルを閉じる

|---|---|---|---|---|

## ユーザ確認事項
- **エンジニア属性の連動**: 「エンジニア属性」をチェックした場合、エンジニアとしての詳細情報（スキルシートなど）の入力フローへの誘導が必要か確認が必要です（現状は属性付与のみ）。
