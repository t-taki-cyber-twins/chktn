# 面談ラウンド管理コンポーネント (app-meeting-round-manager.js)

## 概要
面談回数（1次面談、2次面談など）を管理するコンポーネントです。複数回にわたる面談プロセスをタブ形式で表示し、各回の日時、場所、ステータス、面談ツール、メモなどを管理します。

## 機能の目的
- 複数回の面談プロセスを視覚的にわかりやすく管理する。
- 各面談回の詳細情報（日時、場所、ステータス等）を個別に記録できるようにする。
- 面談の進行状況を一目で把握できるようにする。

## ユースケース
- **面談回の追加**: ユーザーが1次面談の後、2次面談を追加する。
- **面談回の切り替え**: ユーザーが過去の面談回をタブで切り替えて確認・編集する。
- **面談情報の入力**: ユーザーが各面談回の日時、場所、ステータス、メモなどを入力する。
- **ツール設定**: ユーザーがオンライン面談のツール（Zoom等）と参加者を設定する。
- **面談回の削除**: ユーザーがステータス未設定の面談回を削除する。

## EARS 記法による要件定義

### 面談回管理
- **WHEN** ユーザーが「追加」ボタンをクリックした際
  - **THEN** システムは新しい面談回（次の回数）を作成しアクティブにする
- **WHEN** ユーザーが面談回タブをクリックした際
  - **THEN** システムは現在の面談回のデータを保存し、選択された面談回を表示する

### データ保存
- **WHEN** ユーザーが入力フィールドを変更した際
  - **THEN** システムは変更内容を自動保存する
- **WHEN** ユーザーが面談ステータスを変更した際
  - **THEN** システムは親コンポーネントに通知する
  - **AND** 削除ボタンの表示/非表示を更新する

### 削除処理
- **WHEN** ユーザーが「この面談を削除」ボタンをクリックした際
  - **IF** 面談ステータスが設定されている場合
    - **THEN** システムはエラーメッセージを表示し削除を中止する
  - **ELSE**
    - **THEN** システムは確認ダイアログを表示し、承認後に削除イベントを発火する

### ツール設定
- **WHEN** ユーザーが「参加者設定・URL発行」ボタンをクリックした際
  - **THEN** システムは面談ツール設定モーダルを開くイベントを発火する

## ユーザ確認事項
- **削除制御ロジック**: 現在はステータス未設定の面談回のみ削除可能としていますが、この制御ルールが適切かどうか確認が必要です（例：過去の面談を誤って削除しないための制御）。
- **面談時間の選択肢**: 15分から120分まで設定可能ですが、この範囲が適切かどうか確認が必要です。
