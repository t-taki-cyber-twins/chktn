# エンジニア編集 (engineer-edit.html)

## 概要
既存のエンジニア情報を編集するための画面です。
基本情報やスキルの更新だけでなく、そのエンジニアに関連する案件（面談中・稼働中）の一覧表示や、面談情報の追加・編集もここから行うことができます。

## 機能の目的
- 登録済みのエンジニア情報を最新の状態に更新する。
- エンジニアの稼働状況（案件アサイン状況）を把握し、面談調整などのアクションを行う。

## ユースケース
- **エンジニア情報の更新**: ユーザーは、エンジニアのスキルアップや稼働条件の変更に合わせて情報を更新する。
- **関連案件の確認**: ユーザーは、このエンジニアが現在面談中または稼働中の案件を確認する。
- **面談情報の管理**: ユーザーは、新たな案件への面談を追加したり、既存の面談情報を更新したりする。

## EARS 記法による要件定義

### エンジニア情報編集
- **WHEN** ユーザがエンジニア編集画面を開いた際
  - **THEN** システムは既存のエンジニア情報をフォームにロードして表示する
  - **AND** 関連する案件（面談中・稼働中）の一覧を表示する

- **WHEN** ユーザが「更新」ボタンをクリックした際
  - **THEN** システムは入力内容をバリデーションする
  - **IF** バリデーションエラーがない場合
    - **THEN** システムはエンジニア情報を更新する
    - **AND** 完了メッセージを表示する

### 関連案件管理
- **WHEN** ユーザが「+ 面談追加」ボタンをクリックした際
  - **THEN** システムは面談編集モーダル（`app-engineer-meeting-edit`）を新規作成モードで開く

- **WHEN** ユーザが関連案件一覧の「編集」ボタンをクリックした際
  - **THEN** システムは面談編集モーダルを編集モードで開き、該当する面談情報をロードする

## ユーザ確認事項
- **関連案件の表示範囲**: 「表示している自テナントのエンジニアの面談中のものと稼働中の案件」とありますが、過去の稼働履歴や、不採用になった案件を表示する必要があるか確認が必要です。
- **案件登録モーダル**: HTMLには `app-project-register-modal` が含まれていますが、画面上のアクションボタンとしての呼び出し元が明確ではありません（面談追加フローの中で案件を新規登録する場合などに使用される可能性があります）。
