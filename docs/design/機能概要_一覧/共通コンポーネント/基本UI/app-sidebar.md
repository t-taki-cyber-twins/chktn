# サイドバー (app-sidebar.js)

## 概要
アプリケーションの左側に配置されるサイドバーコンポーネントです。各画面固有の検索フォームやナビゲーションメニューを格納するコンテナとして機能し、下部には共通機能である「最近使った画面」を表示します。

## 機能の目的
- **レイアウトの統一**: サイドバーの基本構造（`<aside>`, `<nav>`）を統一し、一貫したレイアウトを提供する。
- **画面固有コンテンツの表示**: 各画面で必要な検索条件入力フォームやメニューを柔軟に配置できるようにする。
- **共通機能の提供**: どの画面からでもアクセスできるように、「最近使った画面」機能を統合する。

## ユースケース
- **検索条件の入力**: ユーザーは、サイドバーに表示された検索フォームに条件を入力し、一覧の絞り込みを行う。
- **最近使った画面へのアクセス**: ユーザーは、サイドバー下部のリストから過去にアクセスした画面へ素早く移動する。

## EARS 記法による要件定義

### コンテンツ表示
- **WHEN** コンポーネントが初期化された際
  - **THEN** システムはコンポーネント内部に定義されたHTMLコンテンツ（検索フォーム等）を保持する
  - **AND** `<aside class="sidebar"><nav>` 内に保持したコンテンツを展開する

### 依存コンポーネントの読み込み
- **WHEN** コンポーネントが接続された際
  - **THEN** システムは `app-recent-screens` コンポーネントが定義されているか確認する
  - **IF** 定義されていない場合
    - **THEN** システムは `app-recent-screens.js` を動的にインポートする

- **WHEN** レンダリングが行われる際
  - **THEN** システムはコンテンツの下部に `<app-recent-screens>` を配置する

## 機能一覧
