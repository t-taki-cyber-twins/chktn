# 最近使った画面 (app-recent-screens.js)

## 概要
ユーザーが最近アクセスした画面の履歴を表示するコンポーネントです。サイドバーの下部に配置され、過去の作業コンテキスト（特定のエンジニアや案件など）へ素早く復帰するためのリンクを提供します。

## 機能の目的
- **作業効率の向上**: ユーザーが直近で作業していた画面へワンクリックで戻れるようにし、ナビゲーションの手間を削減する。
- **コンテキストの保持**: 単なる画面遷移だけでなく、特定のデータ（例：田中太郎さんの面談設定）へのリンクを提供することで、作業の継続性を支援する。

## ユースケース
- **作業の再開**: ユーザーは、別のタスク（メール確認など）を行った後、サイドバーの「最近使った画面」から直前のエンジニア編集画面へ戻り、作業を再開する。
- **頻繁な切り替え**: ユーザーは、複数の案件を並行して確認する際、履歴リストを使って案件間をスムーズに行き来する。

## EARS 記法による要件定義

### 履歴の表示
- **WHEN** コンポーネントが初期化された際
  - **IF** `data-screens` 属性に有効なJSONデータが設定されている場合
    - **THEN** システムは提供されたデータに基づいて履歴リスト（アイコン、タイトル、詳細）を表示する
  - **ELSE** （データがない場合）
    - **THEN** システムはデフォルトのダミーデータを表示する（開発・デモ用）

### データの更新
- **WHEN** `updateScreens` メソッドが呼び出された際
  - **THEN** システムは内部データを新しいリストで更新する
  - **AND** リスト表示を再描画する
