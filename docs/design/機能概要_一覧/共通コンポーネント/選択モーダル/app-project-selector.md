# 案件選択モーダル (app-project-selector.js)

## 概要
自社の案件情報を検索・選択するための共通モーダルコンポーネントです。
エンジニアへの案件割り当てや、各種紐付け操作時に使用されます。

## 機能の目的
- ユーザーが自社の保有する案件の中から、目的の案件を迅速に検索・選択できるようにする。
- 案件の状態（募集中、面談中など）やエンド企業名での絞り込みを可能にし、効率的な選択を支援する。

## ユースケース
- **エンジニアへの案件打診**: エンジニア登録・編集画面にて、そのエンジニアに紹介・割り当てたい案件を選択する。
- **面談設定**: 面談対象となる案件を選択する（案件起点の面談作成時など）。

## EARS 記法による要件定義

### モーダル表示・初期化
- **WHEN** 呼び出し元画面から案件選択モーダルが開かれた際
  - **THEN** システムは自社の案件一覧を表示する
  - **AND** 以前の選択状態や検索条件はリセットされた状態で表示する

### 検索・フィルタリング
- **WHEN** ユーザーが検索条件（案件名、ステータス、エンド企業）を入力して「検索」ボタンをクリックした際
  - **THEN** システムは条件に一致する案件のみをフィルタリングして表示する

- **WHEN** ユーザーが「リセット」ボタンをクリックした際
  - **THEN** システムは検索条件をクリアし、全案件を表示する
  - **AND** 選択中の案件があれば選択を解除する

### 選択・確定
- **WHEN** ユーザーが案件リストの項目をクリックした際
  - **THEN** システムはクリックされた案件を選択状態（ハイライト）にする
  - **AND** 「選択」ボタンを有効化（活性化）する
  - **AND** 他に選択されていた案件があれば選択を解除する（単一選択）

- **WHEN** ユーザーが「選択」ボタンをクリックした際
  - **THEN** システムは選択された案件情報を呼び出し元に返し、モーダルを閉じる

- **WHEN** ユーザーが「キャンセル」または「×」ボタンをクリックした際
  - **THEN** システムは変更を破棄してモーダルを閉じる

## 機能一覧

| コンポーネント | 機能名 | 概要 | 優先度 | アクター | 備考 |
|---|---|---|---|---|---|
| 案件選択モーダル | 案件一覧表示 | 案件をリスト表示する | 必須 | システム | 案件名、エンド企業、ステータスを表示 |
| 案件選択モーダル | 案件検索 | 案件名（部分一致）、ステータス、エンド企業で検索する | 必須 | ユーザー | |
| 案件選択モーダル | 検索リセット | 検索条件と選択状態を初期化する | 必須 | ユーザー | |
| 案件選択モーダル | 単一選択 | リストから案件を1つ選択する | 必須 | ユーザー | 行クリックで選択 |
| 案件選択モーダル | 選択確定 | 選択した案件情報を確定し、呼び出し元へ渡す | 必須 | ユーザー | |

## ユーザ確認事項
- **ページネーション**: 案件数が増加した場合の対応（ページネーションまたは無限スクロール）について確認が必要です。現状は全件取得・クライアントサイドフィルタリングの想定です。
- **選択モード**: 現状は「単一選択」のみですが、将来的に「複数選択」が必要になるユースケースがないか確認が必要です。
- **表示項目**: 一覧に表示する項目（現在は案件名、エンド企業、ステータス）で十分か、他に必要な情報（単価、勤務地など）がないか確認が必要です。
