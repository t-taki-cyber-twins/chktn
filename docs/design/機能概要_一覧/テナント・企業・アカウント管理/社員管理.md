# 社員管理 (employee-list.html)

## 概要
テナントに所属する社員の情報を管理するための画面です。
社員の追加・編集・検索を行い、社員属性（エンジニア、案件担当、技術者管理）やステータス（在籍中、退職済みなど）を管理します。

## 機能の目的
- 社員情報を一元管理し、エンジニア代表や案件担当の選択時に正確な情報を活用する。
- 社員の属性やステータスを適切に管理し、組織の現状を把握する。

## ユースケース
- **社員の追加**: ユーザーは、新しい社員を登録する。
- **社員の検索**: ユーザーは、社員名、メールアドレス、所属部署、ステータス、社員属性で社員を検索する。
- **社員情報の編集**: ユーザーは、既存の社員情報を更新する。
- **ソート機能の利用**: ユーザーは、氏名、所属部署、ステータス、入社日でソートする。

## EARS 記法による要件定義

### 社員検索・一覧表示
- **WHEN** ユーザが社員管理画面を開いた際
  - **THEN** システムは社員の一覧を表示する
  - **AND** 検索フォームを表示する

- **WHEN** ユーザが検索条件（社員名、メールアドレス、所属部署、ステータス、社員属性）を入力して「検索」ボタンをクリックした際
  - **THEN** システムは条件に一致する社員のみをフィルタリングして表示する

- **WHEN** ユーザがカラムヘッダーをクリックした際
  - **THEN** システムはそのカラムでソートする（昇順/降順切り替え）

### 社員管理
- **WHEN** ユーザが「新規社員登録」ボタンをクリックした際
  - **THEN** システムは社員編集モーダル（`app-employee-edit-modal`）を新規作成モードで表示する

- **WHEN** ユーザが「編集」ボタンをクリックした際
  - **THEN** システムは社員編集モーダルを編集モードで表示し、該当する社員情報をロードする

## ユーザ確認事項
- **社員属性の意味**: エンジニア、案件担当、技術者管理の各属性が重複して設定可能か（例：エンジニアかつ案件担当）、それとも排他的か確認が必要です。
- **削除機能**: 画面には削除ボタンが見当たりませんが、社員の削除機能は必要か、または退職済みステータスで管理するのか確認が必要です。
