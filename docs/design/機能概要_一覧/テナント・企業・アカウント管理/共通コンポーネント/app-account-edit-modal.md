# アカウント情報編集モーダル (app-account-edit-modal.js)

## 概要
システム利用者のアカウント情報（ログイン情報、権限、ステータス）を管理するためのモーダルコンポーネントです。社員情報と紐付けを行い、システムへのアクセス権限を制御します。

## 機能の目的
- ユーザー（社員）に対してシステム利用アカウントを発行する。
- 適切な権限（ロールグループ）を割り当て、アクセス制御を行う。
- アカウントの有効/無効やセキュリティ設定（二段階認証）を管理する。

## ユースケース
- **アカウント発行**: 管理者は、新入社員に対してシステム利用アカウントを作成し、初期パスワードを設定する。
- **権限変更**: 管理者は、社員の昇進や異動に伴い、アカウントのロールグループ（権限）を変更する。
- **アカウント停止**: 管理者は、退職した社員のアカウントを無効化し、ログインできないようにする。
- **パスワードリセット**: 管理者は、パスワードを忘れたユーザーのために新しいパスワードを設定する。

## EARS 記法による要件定義

### モーダル表示
- **WHEN** 呼び出し元画面（アカウント一覧など）から「編集」または「新規登録」アクションが実行された際
  - **THEN** システムはアカウント情報編集モーダルを表示する
  - **IF** 新規登録の場合
    - **THEN** パスワード入力フィールドを必須項目とする
  - **IF** 既存データの編集の場合
    - **THEN** システムは現在のアカウント情報をフォームにプレ入力する
    - **AND** パスワード入力フィールドを任意項目（変更時のみ入力）とする

### データ入力・保存
- **WHEN** ユーザーが「保存」ボタンをクリックした際
  - **THEN** システムは入力されたアカウント情報を検証し、保存する
  - **IF** 新規作成時
    - **THEN** 初期パスワード通知メールの送信トリガーを引く（バックエンド処理）

## ユーザ確認事項
- **パスワードポリシー**: パスワードの複雑性要件（文字数、文字種など）についての具体的なバリデーションルールを確認する必要があります。
- **初期パスワード通知**: 画面上の注意書きに「初期パスワードが送信されます」とありますが、具体的な通知手段（メール本文など）の仕様確認が必要です。
