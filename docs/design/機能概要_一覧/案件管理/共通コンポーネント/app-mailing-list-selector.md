# メーリングリスト選択 (app-mailing-list-selector.js)

## 概要
通知対象となるメーリングリスト（ML）を選択するためのモーダルコンポーネントです。
案件やエンジニアの担当者、または社内通知先として、個人のメールアドレスではなくグループアドレス（メーリングリスト）を指定する場合に使用します。

## 機能の目的
- 担当者の異動や退職に影響されない、役割ベースの連絡先管理を実現する。
- 複数の関係者への一斉通知を容易にする。

## ユースケース
- **案件担当者の設定**: 案件の問い合わせ先として、特定の部署やプロジェクトチームのメーリングリストを設定する。
- **社内通知先の設定**: エンジニアの稼働状況や案件の進捗に関する社内通知先として、経営層や営業部のメーリングリストを設定する。

## EARS 記法による要件定義

### メーリングリスト選択
- **WHEN** ユーザがメーリングリスト名検索キーワードを入力して検索ボタンを押した際
  - **THEN** システムはキーワードに一致するメーリングリストをリスト表示する
  
- **WHEN** ユーザがメーリングリストのチェックボックスを選択した際
  - **THEN** システムは選択されたメーリングリストを保持する

### 確定・キャンセル
- **WHEN** ユーザが「選択を確定」ボタンを押した際
  - **THEN** システムは選択されたメーリングリストのリストを呼び出し元に返却する
  - **AND** モーダルを閉じる

- **WHEN** ユーザが「キャンセル」ボタンまたは閉じるボタンを押した際
  - **THEN** システムは変更を破棄してモーダルを閉じる

## ユーザ確認事項
- **メーリングリストの管理**: このコンポーネントで表示されるメーリングリストのデータソース（マスタ管理画面など）についての仕様確認が必要です。現在はモックデータが表示されています。
- **権限による表示制御**: ユーザーの権限によって、選択可能な（または表示される）メーリングリストを制限する必要があるか確認が必要です（例：経営層MLは一般社員には表示しないなど）。
