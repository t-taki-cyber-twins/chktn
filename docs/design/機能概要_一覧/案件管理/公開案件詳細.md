# 公開案件詳細 (public-project-detail.html)

## 概要
他社が公開している案件の詳細情報を確認し、自社のエンジニアを提案（面談申込）したり、問い合わせを行ったりするための画面です。

## 機能の目的
- 公開案件の詳細な内容（スキル要件、条件など）を把握する。
- 自社のエンジニアを提案し、マッチングの機会を創出する。
- 案件に関する疑問点を解消するための問い合わせを行う。

## ユースケース
- **案件詳細の確認**: ユーザーは、案件のスキル要件、単価、勤務地などの詳細情報を確認する。
- **エンジニアの選択**: ユーザーは、この案件に提案したい自社のエンジニアを選択する。
- **面談申込**: ユーザーは、選択したエンジニアで面談を申し込む。
- **問い合わせ**: ユーザーは、案件に関する質問や確認事項を問い合わせる。
- **問い合わせ履歴の確認**: ユーザーは、この案件に関する過去の問い合わせ履歴（スレッド）を確認する。
- **関連エンジニアの確認**: ユーザーは、この案件に既に提案中または稼働中の自社エンジニアを確認する。

## EARS 記法による要件定義

### 案件詳細表示
- **WHEN** ユーザが公開案件詳細画面を開いた際
  - **THEN** システムは案件の詳細情報を表示する
  - **AND** 問い合わせスレッド一覧を表示する
  - **AND** 関連する自社エンジニア（もしあれば）を表示する

### エンジニア提案
- **WHEN** ユーザが「エンジニアを選択」ボタンをクリックした際
  - **THEN** システムはエンジニア選択モーダル（`app-engineer-selector`）を表示する

- **WHEN** ユーザがエンジニアを選択した際
  - **THEN** システムは選択されたエンジニアの情報をサイドバーに表示する
  - **AND** 「面談申込」ボタンを有効化する

- **WHEN** ユーザが「面談申込」ボタンをクリックした際
  - **THEN** システムは確認ダイアログを表示する
  - **IF** ユーザが承認した場合
    - **THEN** システムは面談申込処理を実行し、完了メッセージを表示する

### 問い合わせ
- **WHEN** ユーザが「問い合わせ」ボタンをクリックした際
  - **THEN** システムは問い合わせ入力フォーム（またはモーダル）を表示する（詳細仕様確認中）

- **WHEN** ユーザが問い合わせスレッド一覧の「返信」や「新規作成」を行う際
  - **THEN** システムはメッセージ返信/作成モーダルを表示する

## ユーザ確認事項
- **問い合わせ機能の詳細**: 「問い合わせ」ボタン押下時の挙動（別画面遷移か、モーダルか、メッセージリストへの追加か）について、現状のモックではアラート表示のみとなっているため、具体的なUI仕様の確認が必要です。
- **エンド会社の表示**: 検索画面同様、詳細画面でもエンド会社名が表示されています。公開範囲として適切か確認が必要です。
